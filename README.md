# 基于泰山派 3M 的 Linux 嵌入式人脸识别终端

## 项目背景

本项目面向嵌入式 Linux 应用开发方向，目标是在泰山派 3M 开发板上实现一个可独立运行的人脸识别终端。项目用于积累 Linux 应用层开发、设备调试、系统服务化部署和嵌入式项目工程化经验，并作为后续简历中的 Linux 相关项目。

项目定位不是单纯的人脸识别 Demo，而是一个具备摄像头采集、屏幕显示、人脸录入、人脸识别、日志记录和开机自启动能力的小型嵌入式 Linux 应用系统。前期重点放在 Linux 应用层开发，后期再补充摄像头、屏幕、设备树、驱动加载、内核编译和系统移植等底层知识。

## 项目目标

- 基于 Debian 12 在泰山派 3M 上搭建嵌入式 Linux 人脸识别终端。
- 调通 IMX415 MIPI 摄像头，实现实时图像采集。
- 调通 4.3 寸 MIPI 屏幕，实现本地实时预览和识别结果显示。
- 实现人脸检测、人脸录入、人脸识别和陌生人判断。
- 实现本地人员库、识别记录和运行日志。
- 使用 systemd 完成程序开机自启动和异常退出自动重启。
- 整理部署文档、调试记录、性能数据和可用于简历表达的项目亮点。

## 硬件与开发环境

### 硬件平台

- 开发板：泰山派 3M
- 操作系统：Debian 12
- 显示设备：4.3 寸 MIPI 接口屏幕
- 摄像头：MIPI 接口 IMX415 摄像头

### 开发环境

- 主机系统：Ubuntu 24
- 开发能力：支持本地编译内核或完整系统镜像
- 目标方向：先完成 Linux 应用层开发闭环，再补充驱动、设备树、系统移植相关知识

### 计划使用的技术栈

- 编程语言：C/C++，Python 辅助脚本
- 图像采集：V4L2、OpenCV
- 图像显示：Framebuffer、DRM/KMS、OpenCV 或 Qt，按板端实际显示链路选择
- 人脸识别：OpenCV DNN、ONNX Runtime、NCNN 或平台支持的推理框架
- 数据存储：SQLite 或 CSV
- 服务管理：systemd
- 调试工具：dmesg、v4l2-ctl、media-ctl、journalctl、strace、gdb、top、perf

## 实现流程

### 1. 摄像头链路验证

目标是确认 IMX415 摄像头在 Debian 12 中被正确识别，并能从应用层获取图像。

计划验证内容：

- 使用 `dmesg` 查看摄像头驱动加载日志。
- 使用 `v4l2-ctl --list-devices` 确认视频设备节点。
- 使用 `v4l2-ctl --list-formats-ext` 查看支持的分辨率、像素格式和帧率。
- 使用 `media-ctl` 查看 media pipeline。
- 使用 OpenCV 或 V4L2 测试程序抓取一帧图片。

阶段产出：

- 摄像头设备节点记录
- 摄像头格式和分辨率记录
- 测试抓图
- 摄像头调试文档

### 2. 屏幕链路验证

目标是确认 4.3 寸 MIPI 屏幕可以稳定显示图像，并选择后续应用显示方案。

计划验证内容：

- 确认系统是否启用 framebuffer、DRM/KMS 或桌面环境。
- 显示纯色、测试图片或摄像头预览画面。
- 记录屏幕分辨率、刷新率和旋转方向。
- 确认应用最终通过 OpenCV、Qt、Framebuffer 或 DRM 显示。

阶段产出：

- 屏幕显示方式记录
- 测试显示图片
- 显示链路调试文档

### 3. 实时预览

目标是将摄像头采集和屏幕显示串联起来，形成基础图像通路。

计划实现内容：

- 打开摄像头设备。
- 设置分辨率和像素格式。
- 读取视频帧。
- 转换图像格式。
- 在屏幕上实时显示画面。
- 统计 FPS 和显示延迟。

阶段产出：

- 实时预览程序
- FPS 记录
- 分辨率和格式配置说明

### 4. 人脸检测

目标是在实时画面中检测人脸，并在屏幕上显示检测框。

计划实现内容：

- 接入人脸检测模型。
- 对摄像头帧进行缩放、颜色转换和预处理。
- 执行模型推理。
- 解析检测框。
- 在预览画面中绘制人脸框。

阶段产出：

- 人脸检测模块
- 检测效果截图
- 推理耗时记录

### 5. 人脸录入

目标是支持录入人员信息，并建立本地人脸库。

计划实现内容：

- 输入人员姓名或 ID。
- 抓取当前人脸图片。
- 提取人脸特征。
- 保存人员信息、图片和特征数据。
- 支持人员库加载和查询。

阶段产出：

- 本地人员库
- 人脸录入功能
- 录入数据格式说明

### 6. 人脸识别

目标是实现实时识别闭环。

计划实现内容：

- 对检测到的人脸提取特征。
- 与本地人员库进行相似度比对。
- 根据阈值判断已注册用户或陌生人。
- 在屏幕显示姓名、置信度和识别状态。
- 保存识别记录。

阶段产出：

- 实时识别功能
- 陌生人判断逻辑
- 识别记录
- 识别准确率和耗时记录

### 7. 多线程架构

目标是减少采集、推理、显示和日志之间的阻塞。

建议线程划分：

- 采集线程：负责从摄像头读取视频帧。
- 推理线程：负责人脸检测和识别。
- 显示线程：负责屏幕预览和结果绘制。
- 记录线程：负责日志和识别记录写入。

关键点：

- 使用线程安全队列传递图像帧和识别结果。
- 控制队列长度，避免延迟不断堆积。
- 推理速度低于采集速度时丢弃旧帧，优先保证实时性。

阶段产出：

- 多线程任务模型
- 线程间队列模块
- 延迟和 FPS 对比数据

### 8. 服务化部署

目标是让程序像正式嵌入式应用一样运行。

计划实现内容：

- 编写启动脚本。
- 编写 systemd service 文件。
- 配置开机自启动。
- 配置异常退出自动重启。
- 使用 journalctl 查看运行日志。
- 支持配置文件管理摄像头参数、识别阈值和数据路径。

阶段产出：

- systemd 服务文件
- 安装和启动脚本
- 运行日志
- 部署说明

### 9. 性能优化与记录

目标是形成可以写进简历和面试讲解的量化数据。

计划记录指标：

- 摄像头采集帧率
- 屏幕显示帧率
- 单帧人脸检测耗时
- 单次人脸识别耗时
- CPU 占用
- 内存占用
- 程序连续运行时间

可优化方向：

- 降低输入分辨率
- 推理线程跳帧
- 图像预处理复用缓冲区
- 控制日志写入频率
- 使用更轻量的模型或推理框架

阶段产出：

- 性能测试表
- 优化前后对比
- 性能分析说明

## 面试讲解主线

推荐面试时按以下顺序讲解：

1. 项目是一个运行在泰山派 3M 上的嵌入式 Linux 人脸识别终端。
2. 系统基于 Debian 12，外接 IMX415 MIPI 摄像头和 4.3 寸 MIPI 屏幕。
3. 应用层通过 V4L2/OpenCV 获取摄像头画面，并在屏幕上实时预览。
4. 在预览基础上接入人脸检测和识别，实现人员录入、比对和陌生人判断。
5. 程序内部通过多线程拆分采集、推理、显示和日志写入，保证实时性。
6. 使用 systemd 做服务化部署，实现开机自启动和异常自动重启。
7. 后续补充了解摄像头、屏幕、设备树、驱动加载和 media pipeline，能够解释应用层设备节点背后的系统链路。
